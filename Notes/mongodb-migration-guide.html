üîß Method 1: Simple Migration

Step 1: Set Up MongoDB Atlas

  1. Create an Account

    Visit MongoDB Atlas and sign up for free.

  2. Create a Cluster

    Choose the FREE M0 tier, select your cloud provider and region, then click "Create Cluster".

  3. Create Database User

    Navigate to "Database Access" ‚Üí "Add New Database User"

    Important: Save your username and password - you'll need them for the connection string!
  4. Whitelist IP Address

    Go to "Network Access" ‚Üí "Add IP Address" ‚Üí Allow access from anywhere (0.0.0.0/0) for development.

Step 2: Update Your Connection String

Update your .env file with your MongoDB Atlas connection:

# MongoDB Atlas Cloud Connection
MONGO=mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/yatranepal

# Your old localhost connection (keep for backup)
# MONGO=mongodb://localhost:27017/yatranepal

  • @ ‚Üí %40
  • # ‚Üí %23
  • $ ‚Üí %24
  • Space ‚Üí %20

üîÑ Method 2: Dual-Write Mode (Advanced)

Write to Both Databases Simultaneously

This advanced method allows you to write data to both localhost AND Atlas at the same time, ensuring you have a local backup while using the cloud.

Step 1: Configure Environment Variables

Add these variables to your .env file:

# Primary Connection (MongoDB Atlas)
MONGO=mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/yatranepal

# Secondary Connection (Localhost Backup)
MONGO_LOCAL=mongodb://localhost:27017/yatranepal

# Enable dual-write mode
DUAL_WRITE=true

Step 2: Create Dual Database Utility

Create utils/dualDatabase.js in your API folder:

import mongoose from "mongoose";

let primaryConnection = null;
let secondaryConnection = null;

export const initializeDualConnections = async () => {
    try {
        // Connect to MongoDB Atlas (Primary)
        primaryConnection = await mongoose.connect(process.env.MONGO);
        console.log("‚úÖ Connected to PRIMARY (Atlas)");

        // Connect to Localhost (Secondary)
        if (process.env.DUAL_WRITE === 'true') {
            secondaryConnection = await mongoose.createConnection(
                process.env.MONGO_LOCAL
            );
            console.log("‚úÖ Connected to SECONDARY (Localhost)");
        }
        
        return { primary: primaryConnection, secondary: secondaryConnection };
    } catch (error) {
        console.error("‚ùå Connection error:", error);
        throw error;
    }
};

Step 3: Update Your Server

Modify your index.js to use dual connections:

import { initializeDualConnections } from "./utils/dualDatabase.js";

const startServer = async () => {
    try {
        await initializeDualConnections();
        
        app.listen(PORT, () => {
            console.log(`Server running on port ${PORT}`);
            if (process.env.DUAL_WRITE === 'true') {
                console.log('üîÑ DUAL WRITE MODE ENABLED');
            }
        });
    } catch (error) {
        console.error("Server error:", error);
        process.exit(1);
    }
};
‚úÖ Benefits of Dual-Write Mode:
  • Zero downtime migration
  • Local backup while testing cloud
  • Easy rollback if needed
  • Gradual transition confidence

üÜò Troubleshooting Common Issues

Authentication Failed

  • Verify username and password are correct
  • Check for special characters in password (URL-encode them)
  • Ensure database user has proper permissions

Connection Timeout

  • Check your IP is whitelisted in Network Access
  • Verify your internet connection
  • Try allowing access from anywhere (0.0.0.0/0) temporarily

MongoServerError: bad auth

  • Double-check credentials
  • Reset password in Database Access settings
  • Ensure user has readWrite permissions
